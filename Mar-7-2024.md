# “安装” 通过 zip 发布的软件到 mac（linux 同理）
很多软件不提供 `installer` 而是直接发布打包好的软件，常见的如 `zip` 压缩包。下载解压之后，会发现文件夹内提供了可执行文件，常常会存放在 `bin` 目录。如 `mongodb` 的 `zip` 解压后的 `bin` 目录，目录内包括 `mongod`，`mongos`
等可执行文件。

为了方便的在终端使用这些二进制文件，需要将其目录添加到 `PATH`。直接的做法是，在 `bash` 的配置文件，如 `.bashrc` 中为 `PATH` 追加 `bin` 目录：
```sh
PATH=/path/to/bin:$PATH;
```
但是这样做有个问题：
1. 随着 `bin` 越来越多，`PATH` 的配置会越来越长。
2. 每次新添加 `bin`，都需要手动添加 `bin` 目录，不够智能。

更好的办法是，固定一个 `bin` 主目录，比如 `～/bin`，每当有需要时，在此目录下创建软链接：

```bash
ln -s /path/to/bin/mongod ~/bin/mongod
```

这样，只需要将 `~/bin` 添加到 `PATH` 即可，而且 `bin` 目录下的软链接可以随时删除，不会影响原始文件。当然，如果软件删除了，记得要清理 `～/bin` 目录下的软链接哦。
